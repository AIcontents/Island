# Симуляция Острова

Дискретная по времени, основанная на сетке симуляция экосистемы изолированного острова. Это приложение на основе Java моделирует жизненные циклы и взаимодействия различных видов животных и растений. Каждый организм на острове следует набору правил для таких действий, как перемещение, еда, размножение и смерть, создавая динамичный и развивающийся виртуальный мир.

## Основные Концепции

-   **Сетка Острова**: Мир представляет собой 2D-сетку, где каждая ячейка представляет отдельное местоположение.
-   **Организмы**: Симуляция населена различными типами организмов, включая хищников, травоядных и растения.
-   **Жизненный Цикл**: Симуляция проходит в дискретных временных шагах, или "циклах". В каждом цикле каждый организм получает возможность выполнять действия на основе своего внутреннего состояния и окружающей среды.
-   **Взаимодействия**: Организмы могут взаимодействовать друг с другом. Например, хищники охотятся на травоядных, а травоядные потребляют растения.

## Структура Кода

Проект организован в несколько классов, которые моделируют компоненты симуляции:

-   **`Симуляция`**: Основной класс, который управляет симуляцией. Он содержит игровой цикл и управляет состоянием симуляции.
-   **`Остров`**: Представляет сетку острова и содержит коллекцию объектов `Местоположение`.
-   **`Местоположение`**: Представляет одну ячейку на острове и содержит организмы, находящиеся в этой ячейке.
-   **`Организм`**: Абстрактный базовый класс для всех живых существ в симуляции.
-   **`Животное`** и **`Растение`**: Абстрактные подклассы `Организма`, которые представляют животных и растения соответственно.
-   **`Хищник`** и **`Травоядное`**: Абстрактные подклассы `Животного` для плотоядных и травоядных животных.
-   **Конкретные Классы Животных** (например, `Волк`, `Лошадь`, `Кролик`): Каждый класс расширяет `Хищника` или `Травоядное` и определяет специфическое поведение и атрибуты для этого животного.

### Детализация Тестов

Вот разбивка 13 тестов в наборе:

**`AnimalTest` (2 теста)**
- `testAnimalCreation`: Проверяет, что животное (в данном случае Волк) создается с правильными начальными атрибутами (вес, максимальное количество на ячейку, скорость и сытость).
- `testAnimalCreationBear`: Проверяет то же самое для другого животного (Медведя), чтобы убедиться, что разные типы животных могут быть созданы правильно.

**`LocationTest` (4 теста)**
- `testAddOrganism`: Проверяет, можно ли успешно добавить организм в локацию.
- `testRemoveOrganism`: Проверяет, можно ли успешно удалить организм из локации.
- `testGetOrganisms`: Проверяет, что метод `getOrganisms` возвращает правильный список всех организмов в локации.
- `testAddOrganismExceedsCapacity`: Проверяет, что невозможно добавить в локацию больше организмов одного вида, чем это разрешено его атрибутом `maxPerCell`.

**`IslandTest` (4 теста)**
- `testIslandCreation`: Проверяет, что остров создается с правильными размерами (ширина и высота).
- `testGetLocation`: Проверяет, можно ли получить объект `Местоположение` по его координатам.
- `testGetAllLocations`: Проверяет, что метод `getAllLocations` возвращает полный список всех локаций на острове.
- `testInvalidLocation`: Проверяет, что попытка доступа к локации за пределами острова вызывает ошибку `IndexOutOfBoundsException`.

**`SimulationFunctionalTest` (2 теста)**
- `testSimulationRun`: Запускает короткий цикл симуляции и проверяет, что симуляция выполняется без сбоев и основные объекты (как остров) остаются доступными.
- `testPredatorEatsHerbivore`: Функциональный тест, который проверяет один из ключевых механизмов взаимодействия: хищник (Волк) съедает травоядное (Лошадь), когда они находятся в одной ячейке.

**`DemoApplicationTests` (1 тест)**
- `contextLoads`: Стандартный тест Spring Boot, который проверяет, что контекст приложения успешно загружается без ошибок.
